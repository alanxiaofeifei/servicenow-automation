# 3–5 分钟演示脚本

## 1. 开场

服务台工程师每天要把零散的支持上下文整理成标准 ServiceNow 工单。这个 demo 展示的是一个 human-in-the-loop workbench：把手动粘贴的问题描述转换成可编辑 Incident 草稿，匹配本地知识库，并填入安全的 Mock ServiceNow 表单。

## 2. 安全声明

这不是自动提交机器人。工具只自动化草稿整理，不转移责任。最终提交仍然必须由人工完成。

## 3. VPN 场景

1. 点击 **Load VPN Demo**。
2. 展示原始粘贴文本。
3. 展示生成的 TicketDraft。
4. 指出 Short Description、Description、Work Notes、Category、Subcategory、Assignment Group、Priority。
5. 展示 VPN KB match。
6. 手动编辑一个字段，证明服务台人员仍然掌控结果。
7. 展示 Risk Control Gate。
8. 在填表前确认 human review。
9. 展示 Mock ServiceNow Incident form。
10. 指出 submit 在 demo mode 下禁用。

## 4. 快速展示另外两个场景

快速切换：

- Windows Demo
- Account/Login Demo

说明同样流程可以覆盖常见服务台 triage 场景。

## 5. 业务价值句

这个工具的价值是减少重复写工单的时间，提高字段选择和 work notes 的一致性，并让升级交接更安全。

## 6. 面试收尾句

This project demonstrates how I combine real service desk workflow understanding with practical AI-assisted automation design while respecting ITSM control boundaries.
