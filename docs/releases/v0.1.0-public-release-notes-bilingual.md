## English

### What this release is

This is the first public test preview of **ServiceNow Automation Workbench**, a local-first desktop workbench for service-desk operators.

It helps turn sanitized support context into:

- a structured Incident draft;
- local KB recommendations;
- a guided review path;
- a monthly Excel fill-queue placeholder;
- safe browser-assistance boundaries where the human remains responsible for final ServiceNow actions.

The operating rule is:

> AI drafts and helps with allowed text fields. The human reviews and submits.

### Basic usage

1. Download the Windows ZIP.
2. Extract it to a local Windows folder.
3. Read `START-HERE-WINDOWS.txt` first.
4. Double-click `ServiceNow Automation.exe`.
5. Use local demo data or sanitized manual input to review the workbench flow.
6. Do not paste real customer data, real ticket content, production URLs, cookies, sessions, HAR files, traces, or credentials.

### Current capabilities

- Local demo intake queue and manual paste flow.
- Incident draft with short description, description, and work notes.
- Local KB recommendation cards with evidence and suggested support group.
- Guided review path for the operator workflow.
- Monthly Excel fill queue placeholder for post-ticket bookkeeping.
- English and Chinese UI modes.
- Windows ZIP package for manual desktop testing.

### Current limitations

This is a **test preview**, not a final production automation system.

It does not perform:

- real ServiceNow login automation;
- real ticket submission;
- Save / Submit / Update / Resolve / Close automation;
- Microsoft Graph or Excel Web writeback;
- real Teams / Outlook / phone-system ingestion;
- processing of real customer or production ticket content.

### Development vision

The long-term vision is a safe **Service Desk Workflow Cockpit** that helps support agents normalize intake, draft tickets, recommend KB steps, prepare handoff notes, and maintain reporting cues — while keeping final ITSM actions under human control.

## 中文

### 这个版本是什么

这是 **ServiceNow Automation Workbench** 的第一个公开测试预览版。它是一个本地优先的桌面工作台，面向 Service Desk / IT Support 操作者。

它可以把脱敏后的支持上下文整理为：

- 结构化 Incident 草稿；
- 本地 KB 推荐；
- 引导式审核路径；
- 月度 Excel 填写队列占位流程；
- 安全边界内的浏览器辅助流程，并保持最终 ServiceNow 操作由人工完成。

核心原则是：

> AI 可以草拟内容并辅助允许的文本字段；最终审核和提交由人工完成。

### 基础用法

1. 下载 Windows ZIP。
2. 解压到本地 Windows 文件夹。
3. 先阅读 `START-HERE-WINDOWS.txt`。
4. 双击 `ServiceNow Automation.exe`。
5. 使用本地 demo 数据或手动粘贴的脱敏文本查看工作台流程。
6. 不要粘贴真实客户数据、真实工单内容、生产 URL、cookies、sessions、HAR、traces 或凭据。

### 当前功能

- 本地 demo intake queue 和手动粘贴流程。
- Incident 草稿：短描述、描述、Work notes。
- 本地 KB 推荐卡片：证据、置信度、推荐支持组。
- 引导式审核路径。
- 月度 Excel 填写队列占位流程。
- 英文 / 中文界面模式。
- Windows ZIP 桌面测试包。

### 当前限制

这是 **测试预览版**，不是最终生产自动化系统。

它不会执行：

- 真实 ServiceNow 登录自动化；
- 真实工单提交；
- Save / Submit / Update / Resolve / Close 自动化；
- Microsoft Graph 或 Excel Web 写入；
- 真实 Teams / Outlook / 电话系统接入；
- 真实客户或生产工单内容处理。

### 最终开发愿景

长期目标是一个安全的 **Service Desk Workflow Cockpit**：帮助支持人员整理 intake、生成工单草稿、推荐 KB 步骤、准备交接说明和月度记录提示，同时保持最终 ITSM 操作由人工控制。

---

## Safety boundary

No real ServiceNow login, no real ticket submission, no Save / Submit / Update / Resolve / Close automation, no Microsoft Graph / Excel Web writeback, and no real customer data are included in this test preview.
